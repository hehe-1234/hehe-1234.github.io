公司产品介绍

基于 Spark 和 Delta-Lake 之上，实现了湖仓一体的查询引擎
基于 K8S 部署，扩展了 Spark，增加了很多自己的扩展规则，可对接多种数据源
支持 Hive JDBC、REST API 访问，支持多种客户端工具
为公司其他产品提供底座支持

工作内容

总体架构

按模块划分

  • 扩展了 Spark，增加了元数据管理、对接各种数据源，以及自定义的一些优化规则
  • 按照集群的规模分为小、中、大版本，配合资源控制实现隔离机制
  • 可以自己管理存储，存储格式为 Delta-Lake，也可以将数据存储在外部

架构

  • 参考了 Kyuubi，使用了两层调度，实现可高可用
  • 不同集群被划分到 不同的 k8s namespace 下
  • 支持权限认证，SQL 下推

使用的技术

Spark、Delta-Lake

第 1 阶段：统一查询引擎


包括了接入层、解析层、引擎层，【更详细的项目介绍】

  • 接入层提供一个TCP服务，供客户端调用
  • 解析层是最重要的一层，在这一层里会将接入层获取到的查询信息进行分析，之后交给引擎层
  • 引擎层根据解析层的指示，选择spark、flink或者JDBC直连的方式进行查询
  • 引擎层最终调用的就是一个个具体的存储服务

主要工作

  • 整体架构参考了 QuickSQL，【详细调研】，使用 Calcite 做 SQL 解析，【一些记录】
  • 负责架构设计，关键模块设计，项目整体进度推进

第 2 阶段：湖仓一体项目

主要工作

  • 设计并开发了元数据管理模块，通过扩展 Spark 实现自定义元数据功能、高可用模块、资源隔离模块
  • 设计开发 HiveServer 功能，客户端可以用 Hive JDBC 连接
  • 设计开发 高可用功能，这块是参考了 Kyuuib 的架构，重新设计的
  • 设计开发 资源隔离模块，增加产品稳定性和多租户隔离性
