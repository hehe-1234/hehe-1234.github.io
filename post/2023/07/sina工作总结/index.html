公司产品介绍

仿造谷歌 GAE 做的一款 PAAS 产品，是国内最早的 PAAS 产品
产品主页

主要客户

  • 传媒类:新浪娱乐，成都商报，河北卫视
  • 社交类:内推网，YOU+青年社区
  • 金融类:中信银行，东方证券
  • 移动类:天气通，掌上贵金属

工作内容

总体架构

架构如下：

  • 基于物理机之上，部署多个 JVM，每个 JVM 之上再运行一个 Jetty，Jetty 上面部署了多个用户的应用
  • 隔离策略包括了多层，每个用户单独账号读写路径分离
  • 使用 cgroup 控制资源，使用 LD_PRELOAD 给系统库打桩，JVM 方面使用 Java 自己的安全管理器
  • 再往上层的 Jetty 定义了一堆安全策略，同时增加一些门面类，通过禁止对这些门面类调用放射，隔离用户和 Jetty
  • Jetty 中有平台的 SDK，通过 SDK 来访问各种服务
  • 运行期间会日志写入到 日志中心，将监控信息写入到监控中心，调用信息汇总到计费中心，用户 WAR 通过分布式代码管理系统导入
  • 多个 APP 由 Keeper 来管理，负责启动/停止等

使用的技术

第 1 阶段：Java 平台搭建

这个阶段是从 0 开始搭建 Java 平台，并推广试用
主要工作内容

  • 编写 Java 版本的几个核心客户端，实现最基本的 Java 平台环境
  • 做隔离，包括系统级别的用户隔离，基于 Java 安全管理器的隔离
  • 增加 Jetty 的扩展对接 Syslog，增加 Jetty 的 Facade
  • 跟阿里白帽子的几次攻防战，最后修改openjdk解决所有安全问题
  • 企业内部推广，对外推出邀请码试用

第 2 阶段：对外开放

这个阶段已经完全放开注册了，开始接一些大客户支持，增加更多功能
主要工作内容

  • 增加 Jetty 的分布式 Session 功能
  • 关注用户反馈 BBS 解决相关问题，做一些大客户支持
  • 将 KV 存储服务，从 Memcached 迁移到 HBase 并负责维护
  • 竞品调研

第 3 阶段：稳定性改进

因为扩张太快导致平台出现各种稳定性问题，后面不断提升平台稳定性
出现的各种故障

  • 整个机房停电没做异地容灾出现数据丢失问题(KV存储 和 MySQL)
  • GoDaddy 域名续费问题，新浪法务没有及时通知导致一天整个平台域名都没法访问
  • 各种攻击事件，乌云在我们平台上然后整个平台频繁被攻击最后乌云迁走了
  • 因为雾霾导致GPS时钟失灵，机房所有的机器恢复到格林威治初始时间 1970 年
  • 物理机房交换器部署不当，接入层交换器流量过大 buffer 不够了，导致定期出现丢包
  • 运维没有良好的自动运维工具，运维脚本，很多是手动处理的，也导致了很多故障

主要工作

  • 完善整个打包部署上线流程，完善测试用例、自动化测试，增加自动化运维，SLA 最后达到 99.98%
  • KV 存储服务，增加 thrift 接口，可以对外其他语言
  • 尝试转向 Docker
  • 研究 Agent 探针工作开始平台的 APM 监控运维工作
  • 参与可信云认证、大数据联盟项目
  • 将工作上的一些创新发明申请了专利

主要工作业绩和贡献

主要业绩

  • 作为核心人员，参与了从无到有的 Java 平台发展过程，是 Java 平台的最主要贡献者
  • 维护了 KV 存储系统，保证了系统的稳定性
  • 在经历了 10 年一见的故障后，找回了很多丢失数据，帮助平台挽回很大损失
  • 工作期间有过两个创新的技术，帮助平台实现了易用性和稳定性，后将这两个创新申请了专利
